## How to Verify Features

### Srcset Verification
1. Open browser DevTools (F12)
2. Inspect any image above
3. Check the `srcset` attribute - it should contain: `400w, 800w, 1200w, 1600w`
4. The browser loads different sizes based on viewport and DPR

### Blur Loading Verification
1. Throttle network to "Slow 3G" in DevTools
2. Reload the page
3. You should see a blurred preview image while the full image loads
4. The blur fades out as the main image fades in

### Click-to-Enlarge Verification
1. Click any image in the "Different Width Values" or "Carousel" sections
2. A modal opens with the full-resolution image
3. **Close via close button OR clicking the backdrop**
4. Press Escape key to close modal
5. The modal image is generated via `getImage()` without width constraints

### Marquee Verification
1. The marquee above should auto-scroll smoothly
2. Images have blur preview while loading
3. Hover to pause the animation
4. Click-to-enlarge is disabled in marquee for smooth scrolling

## Technical Details

**All components use React** for coherent state management with Astro wrappers for image processing.

**Img_better Features:**
- **Required props**: `src` and `alt` (TypeScript enforced)
- **Blur loading**: 20px wide preview with CSS blur while main image loads
- **Srcset**: Auto-generated `[400, 800, 1200, 1600]` for imported images
- **Percentage widths**: CSS override prevents Astro's inferred dimensions
- **Modal**: Full-size image via `getImage()`, close via button, backdrop click, or Escape key

**Marquee_better Features:**
- React-based with CSS keyframes animation
- Pause on hover support
- Use `enableClickToEnlarge={false}` for smooth scrolling
- Blur loading and srcset still work

**Carousel_better Features:**
- React hooks for state (currentSlide, isPaused)
- Works seamlessly with click-to-enlarge
- Each slide can have independent modal
- Auto-scroll pauses on hover
